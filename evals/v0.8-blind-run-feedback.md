## What I Learned

v0.8 is strongest when the prompt clearly falls into one of these modes:

- multiple-choice intake
- proof gate
- critique mode
- validation stage
- clean pass with supplied proof

It is weaker when the prompt asks for navigation, simulation, launch sequencing, category memo discipline, or rival comparison. In those cases it often produces something strategically useful but misses the exact operating protocol.

The main issue is not taste. The outputs mostly sound good. The issue is that the skill sometimes optimizes for helpfulness over the behavioral contract. That is dangerous for evals because a useful answer can still be the wrong answer.

## Recommended v0.8 Fixes

1. Add a stricter `Stage Control` rule:
   - If user asks `what next?`, answer with current stage, one next action, and missing input only.
   - Do not restart the whole method.

2. Add a `Multi-Turn Simulation` rule:
   - If prompt asks for a 2-turn or 3-turn arc, preserve each turn explicitly.
   - Do not collapse to final output.

3. Add a `Naming Mode` template:
   - candidate
   - tag: explain / compress / provoke
   - thesis
   - risk

4. Add a `Category Memo` template:
   - explain/compress/provoke test
   - what it replaces
   - market question
   - adjacent players
   - category-of-one risk
   - missing proof

5. Add a hard `Rival Check` gate:
   - fetch/read actual rival copy, or ask user to paste it
   - do not score differentiation from memory or assumptions
   - keep rival notes short

6. Strengthen the thin-input rule:
   - if substitute and sacrifice are unknown, interview before writing copy

7. Normalize all mode closings:
   - even light mode needs a one-line `Next:` close
