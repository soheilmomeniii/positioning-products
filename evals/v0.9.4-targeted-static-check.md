# v0.9.4 Targeted Static Check

Checked at: 2026-07-14 21:32 UTC

This is not a blind model re-run. It is a static check after patching the first failed targeted checkpoint from the v0.9.3 candidate run.

## What Changed

The first targeted checkpoint, `vague-ai-trader-tool`, failed because the skill still wrote polished headline-style output from thin input and did not force sacrifice.

v0.9.4 candidate adds:

- Thin copy requests with fragments like "AI tool for traders" must not draft polished artifacts. They get a provisional diagnostic frame plus questions only.
- Thin-input first-pass questions must cover customer, substitute, proof, and sacrifice. If only two questions fit, proof and sacrifice win. If sacrifice is missing, one question must force it.
